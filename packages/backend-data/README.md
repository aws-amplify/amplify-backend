# Description

This package contains the `defineData` entry point for customers to define data in their Amplify backend.
This is a layer on top of the data construct in `@aws-amplify/data-construct` to initialize the construct in the context of an Amplify backend.

## PostgreSQL Provider Support

This package supports PostgreSQL-backed data models via pluggable database providers. Providers are configured through the `database.provider` option in `defineData()`.

### Available Providers

| Provider   | Use Case               | Provisioning                           |
| ---------- | ---------------------- | -------------------------------------- |
| `aurora()` | Aurora Serverless v2   | Yes — creates VPC, cluster, networking |
| `rds()`    | Existing RDS instances | No — connection URI only               |

### Usage

```typescript
import { defineData, aurora } from '@aws-amplify/backend-data';

export const data = defineData({
  schema,
  database: {
    provider: aurora({
      provision: {
        databaseName: 'mydb',
        minCapacity: 0.5,
        maxCapacity: 1,
      },
    }),
  },
  authorizationModes: {
    defaultAuthorizationMode: 'apiKey',
    apiKeyAuthorizationMode: { expiresInDays: 7 },
  },
});
```

### Architecture (Aurora with provisioning)

```
Client → AppSync → SQL Lambda (isolated subnets) → Aurora Serverless v2
                        ↓
                   VPC Endpoints → SSM (reads connection URI)
```

Key implementation details:

- **VPC**: Created with `natGateways: 0` to avoid NAT costs. Uses isolated subnets for Lambda and VPC endpoints for SSM/KMS access.
- **Security group**: Shared between Lambda, VPC endpoints, and Aurora with self-referencing ingress rules for ports 443 (HTTPS) and 5432 (PostgreSQL).
- **Connection URI**: Stored as SSM SecureString parameters at Amplify-standard paths generated by `ParameterPathConversions`. The SQL Lambda reads these at runtime.
- **Subnet selection**: The factory uses `isolatedSubnets` when `privateSubnets` is empty (which occurs when `natGateways: 0`).

### Source Layout

- `src/providers/aurora_construct.ts` — CDK construct for Aurora Serverless v2 + networking
- `src/providers/aurora_provider.ts` — Provider class and `aurora()` factory function
- `src/providers/rds_provider.ts` — Provider for existing RDS instances
- `src/providers/types.ts` — Shared types and type guards
- `src/factory.ts` — `defineData()` with provider integration (subnet selection, VPC config extraction)
